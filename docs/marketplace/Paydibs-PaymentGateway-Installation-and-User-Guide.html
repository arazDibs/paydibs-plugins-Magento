Installation

Requirements

  • Adobe Commerce or Magento Open Source 2.4.x — requires magento/framework ~103.0.0||~104.0.0 (see composer.json).
  • PHP — must match your Magento version and satisfy require.php in the extension composer.json.
  • Marketplace — listing version must match version in composer.json inside the uploaded ZIP.

Install via Composer (recommended)

Magento Open Source

composer require paydibs/module-paymentgateway
bin/magento module:enable Paydibs_PaymentGateway
bin/magento setup:upgrade
bin/magento setup:di:compile
bin/magento cache:flush

Adobe Commerce / Commerce Cloud

composer require paydibs/module-paymentgateway-commerce
bin/magento module:enable Paydibs_PaymentGateway
bin/magento setup:upgrade
bin/magento setup:di:compile
bin/magento cache:flush

For a path repository, point Composer at the folder that contains this package’s composer.json.

Install from ZIP (Marketplace)

  1. Unzip so the root of the folder contains composer.json, registration.php, etc/, and the rest of the module (no extra wrapping directory in path references).
  2. Add that folder as a Composer path repository, then composer require the package; or install through Marketplace when available.
  3. Run the same bin/magento commands as above.

Prefer installation through Composer into vendor/. Copying only to app/code is possible for testing but is not the primary documented flow.

Verify installation

bin/magento module:status Paydibs_PaymentGateway

Expected: Module is enabled.

Admin access

Configuration: Stores → Configuration → Sales → Payment Methods → Paydibs Payment Gateway. Administrators need access to payment settings (Magento Magento_Payment::payment ACL).

After installation

Set Merchant ID, Merchant Password, and API Environment (Test vs Production) before taking live traffic.

User guide

Overview

The extension adds Paydibs as a payment method. Checkout creates an order in pending payment, then redirects the customer to Paydibs. Orders are updated from return URLs, IPN notifications, optional cron, and optional manual requery.

Configuration (Admin)

Stores → Configuration → Sales → Payment Methods → Paydibs Payment Gateway

FieldDescription
EnabledTurn the method on or off.

API EnvironmentTest or Production.

Merchant PasswordFrom Paydibs; stored encrypted.

Enabled LogYes writes to var/log/paydibs.log (no full signatures/secrets).
Restore Cart on Failed PaymentYes attempts quote restoration after failure where applicable.
Manual Requery Secret KeyOptional. If set, GET paydibs/payment/requery?key=<secret> runs the pending-order query (JSON). Empty = disabled.

Checkout flow (customer)

  1. Customer selects Paydibs and places the order.
  2. Browser redirects to Paydibs using the prepared URL.
  3. Paydibs redirects back; order status updates from signed parameters.

Operational features

  • Cron — Job paydibs_query_pending_orders, schedule */15 * * * * (about every 15 minutes). Requires Magento cron to be active.
  • Manual requery — Optional secret-protected GET for support or automation.

Security and privacy

  • Response and Notify validate signatures per Paydibs integration rules.
  • Secrets use Magento encrypted config where applicable.
  • Hosted payment data is processed by Paydibs; align your privacy policy and agreements accordingly.

Support

Paydibs merchant support for gateway accounts. Extension: URLs in composer.json (support.source, support.issues).
