Paydibs Payment Gateway — Adobe Commerce

On-prem & Commerce Cloud · Composer: paydibs/module-paymentgateway-commerce · Adobe Commerce 2.4.x

Who this is for

This guide is for teams running Adobe Commerce (on-premises or Commerce Cloud). Checkout and admin behavior match standard Magento patterns—Paydibs is a payment method with hosted pages and server notifications behind the scenes.

Requirements

  • Adobe Commerce 2.4.x compatible with the package composer.json.
  • PHP per your Adobe Commerce release.
  • Paydibs Test credentials for QA; Production when you launch.

Installing with Composer

From the Magento project root:

composer require paydibs/module-paymentgateway-commerce
bin/magento module:enable Paydibs_PaymentGateway
bin/magento setup:upgrade
bin/magento setup:di:compile
bin/magento cache:flush

On Commerce Cloud, add the dependency the way you manage other extensions: commit composer.json / composer.lock and deploy through your pipeline so vendor/ is built on each environment.

Verify: bin/magento module:status Paydibs_PaymentGateway should show enabled.

Configuration

Use Stores → Configuration → Sales → Payment Methods → Paydibs Payment Gateway. Set scopes (website / store view) the same way you do for other payment methods—e.g. Test on staging, Production on live.

Operations and order view

Ensure cron runs in each environment where you take payments; pending orders are rechecked about every 15 minutes. IPN endpoints must be reachable from Paydibs according to your Adobe Commerce / Cloud networking setup.
